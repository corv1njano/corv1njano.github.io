C#-Syntax verstehen

Artikel erstellt: 26.09.2024 • Artikel zuletzt überarbeitet: 30.09.2024 • 1 mal überarbeitet

Damit ein Programm in C# gültig ist und erfolgreich ausgeführt werden kann, wird eine Klasse und eine Methode benötigt, die das Programm startet. In C# und vielen anderen Programmiersprachen ist die Main-Methode die Methode, die zuerst ausgeführt wird. Das Wort “Main” ist reserviert und kann in jedem C#-Programm nur einmal verwendet werden.

Klassen

Folgender Code zeigt eine leere Klasse in C#. Klassen können Attribute und Methoden beinhalten.

Leere Klasse

Kopieren

class KlassenName

{

  // Attribute und Methoden...

}

Methoden

Folgender Code zeigt eine leere Methode in C#. Methoden werden immer in Klassen deklariert. Methoden können zusätzliche Parameter erhalten. Diese werden, falls gegeben, in der Klammer dahinter notiert.

Leere Methode

Kopieren

static void MethodenName(Parameter...)

{

  // Befehle...

}

Zusatz: Namespaces

Namespaces (Namensbereiche) sind in C# optional, aber sehr nützlich, um den Code klar zu strukturieren. Je mehr Dateien, Klassen und Methoden ein Programm hat und je komplexer es wird, desto schwieriger wird es, den Überblick zu behalten. Daher ist es ratsam, von Anfang an Namespaces zu verwenden. So gewöhnt man sich an ihre Nutzung und weiß, wie man sie effektiv einsetzt, was besonders hilfreich ist, wenn das Projekt wächst. Der Namesapce wird um unsere Klassen und deren Methoden aufgebaut:

Namespaces

Kopieren

class KlassenName

{

  // Attribute und Methoden...

}
